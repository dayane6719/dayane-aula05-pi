##  Criação do repositório

É o processo de criar um espaço no GitHub onde o projeto será armazenado. Ele serve para guardar arquivos, controlar versões e permitir colaboração.

##  Clone do repositório

É quando copiamos o repositório do GitHub para o nosso computador, permitindo trabalhar localmente.

##  Criação de branch

Branch é uma ramificação do projeto principal. Serve para desenvolver novas funcionalidades sem afetar o código principal.

##  Registro de commits

Commit é o ato de salvar alterações no projeto com uma mensagem descritiva, permitindo acompanhar mudanças.

##  Envio das alterações (push)

É quando enviamos as alterações feitas no computador para o GitHub.

##  Atualização do repositório local (pull)

Serve para atualizar o projeto local com as mudanças feitas no GitHub.

##  Criação de Issues

Issues são usadas para registrar tarefas, bugs ou melhorias no projeto.

##  Organização com Milestone

Milestones agrupam issues e ajudam a organizar metas do projeto.

##  Gerenciamento com Project

Projects são quadros (tipo Kanban) para organizar tarefas visualmente.

##  Abertura de Pull Request

Pull Request é quando solicitamos que nossas alterações sejam revisadas e adicionadas ao projeto principal.

##  Realização de Merge

Merge é o processo de juntar as alterações de uma branch na branch principal.
